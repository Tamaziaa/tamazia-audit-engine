## (a) THE INTENDED ARCHITECTURE OF THE AUDIT ENGINE

### The eight-stage pipeline (Spec Ch.1)
A single input — a domain (+ optional sector/country hints, lead_id) — becomes a rendered legal audit through eight stages. Orchestrator: `engine/src/skills/S025-audit-page-builder/scripts/build.js`. Compliance brain: `engine/src/skills/S008-personalisation-engine/scanners/compliance.js → scan()`. Render: `website/functions/audit/_adapter.js → payloadToD()`.

1. **CRAWL** — `compliance.js gatherCorpus()`: up to 120 pages, 28s budget, concurrency 12; policy/fees/checkout prioritised; Jina + Wayback fallbacks; `crawl-escalation.js maybeEscalateCrawl()` → Apify (default off). Out: `corpus[] {url, body, status}` + segment index.
2. **PROFILE** — `audit/firm-profile.js profileFirm()` (LLM own-vs-client, temp 0, Cloudflare→Groq→Gemini) + `sourcing/markets.js detectMarkets()` + `mergeJurisdictions()` two-signal gate. Out: sector, sub?, hq_country, office_countries[], serves[], detected_jurisdictions[].
3. **SIGNALS** — `compliance/signals.js buildSignals()`: `toCanonicalJurisdictions()` (JUR_MAP), `normalizeSector()` (20 SECTOR_RX buckets), `deriveTriggers()` (~23 applies_when flags), `augmentFreezones()` (DIFC/ADGM).
4. **ATTACH** — `compliance/connect.js connect()` (LIVE, called at `compliance.js:805`): GATE A jurisdiction → GATE B0 framework-sector → CAP_GATE capability → GATE B rule sector_relevance → GATE C trigger. Fallback if `connect()` throws: `routeJurisdictions()`. **Orphan (not shipped):** `resolver.js resolveLaws()` — the most rigorous engine, used only by QA `gap-finder.js`.
5. **DETECT** — `loadRules({frameworks})` from Neon `compliance_rules`; `ruleCheck()` runs 4 rule types (must_appear, trigger_then_check, prohibit, element_checklist) over the corpus; `_extractQuote()` + `_absenceEvidence()`.
6. **FILTER** — `resolver.js overlayDrop()` applied **twice** (in `compliance.js` and again as build.js fail-closed guard ~L297); drops unservable/vacated/out-of-jurisdiction/freezone/out-of-sector/below-threshold. Drops are traceless (`dropped[]` never rendered).
7. **ENRICH + ASSEMBLE** — `enforcement-map.js enforcementFor()`; join `framework_intelligence`; penalty exposure; HMAC-sign; write R2 + `audit_pages`.
8. **RENDER** — `_adapter.js payloadToD()`: group-by-framework, sibling-collapse, `fwRegulator()`, `craftFix()` (render-side fix re-derivation — a bug), penalty banding, pillar routing.

### The core disease (the "one fact")
Three parallel implementations of "which laws bind": `connect()` (live), `resolveLaws()` (orphaned), `routeJurisdictions()` (fallback) — the most rigorous does not ship. Compounding it, two hand-maintained predicate vocabularies: the catalogue gates on **113 predicates**, `signals.js` produces **23**, so **102 are unproducible** and silently suppress their laws (the entire US layer, `serves_us_users`×20, first). Four knowledge stores joined by strings: Neon `framework_versions` (~285) + `compliance_rules` (~640) + `compliance-laws.json` (187 laws) + `enforcement-map.js` MAP (~32-40), joined on `framework_short`. Four sector vocabularies across seven files. V2 revised this **worse** than V1: four vocabularies, four enforcement stores, double-applied filter with divergent inputs, stale name map shadowing the catalogue, a parallel LLM compliance-claim path.

## (b) THE 500 LOGIC-INTEGRITY CHECKS

`Tamazia_Audit_Logic_Integrity_500.html` = **500 cascade invariants across 50 scenarios** (10 invariants each). It is the consistency contract, not architecture: given the unified engine, the fifty ways an input can shift and the ten things that must re-derive in lockstep each time. **Governing principle (the "currency example," generalised):** when one fact changes, every dependent fact re-derives from the same single source, the cache key changes so no stale audit is served, and the change is provable on the golden set. An invariant breaks the moment one dependent keeps an old value after its driver moved.

## (d) COWORK BIBLE V3 RULES THAT CONSTRAIN ANY REBUILD

V3 is the operational build manual. Scope hard-bounded: **compliance & regulatory intelligence engine only; ten priority sectors only; UK/EU/USA/Middle East.** It adds the unified legal-dataset record (`one law = one row`), authoritative sourcing feeds (UK/EU/US weekly, ME fortnightly), a module connection matrix, 7 uniformity-enforcement points (UP-1…UP-7), a 10-row uniformity index (UI-1…UI-10), and **300 enumerated items** (100 system truths + 100 accuracy guardrails + 100 logic improvements). Governing constraints for any rebuild:

- **Selection rule that prevents frivolous attachment:** a sector's frameworks attach only when the **sub-sector node matches AND the nexus holds** — never on a bare sector bucket. This is why ABI never reaches a bank, the POM ad ban never reaches a GP surgery, EU consumer law never reaches a pure-B2B consultancy.
- **Legal-correctness guardrails (Cat.1):** never render a finding without real evidence (verbatim quote or true named-element absence); an obligation is a substantive duty, never a statute-section-number recital; a voluntary trade body renders as a voluntary code, never binding law; penalty headlines use the calibrated typical range, never the raw statutory max; where no breach exists, render the 5-field intelligence block, never invent one; "if you would not assert it in a tribunal, do not render it"; doubtful legality → set the rule inactive pending legal review.
- **Evidence discipline (Cat.4):** synthetic URLs forbidden; a quote is attributable to one page (never merge across pages); testimonial claims are not the firm's claims; an absence is valid only if the proving page was crawled (else render **screened**, not breached); evidence + duty + binding-status + fix travel as one typed object — **the render formats, never re-derives**.
- **Determinism (Cat.9):** coverage contract guarantees required pages crawled; cache golden corpora; same firm + same version → same audit; `overlayDrop` runs once with one registry-built sig; one attachment engine fail-closed; every drop recorded and surfaceable; shadow-compare new vs old before any cut-over.
- **No-hallucination boundaries (Cat.10):** detection stays deterministic — an LLM never decides whether a law is breached; the LLM is confined to profiling, offline fix authoring, and outreach pointers; the pointer-generator consumes validated findings and never re-derives a law (a cold email must never contradict the audit — V2 N-5); every change touches the registries, not a local copy. Final rule #100: "Nothing ships that a regulator, a lawyer, or a billionaire prospect could fault in three seconds."
- **Key system truths (already-built capability to wire, not rebuild):** the `active` column quarantines a citation-presence rule with a one-line UPDATE (truth 13); `enforcement.js` already builds the calibrated 5-part penalty panel and already rejects the raw statutory max as a "fake metric" — just not wired to the headline (truths 51–54); the hallucination-guard already requires every pointer to cite an on-scan anchor (truth 45); `craftFix` L648/L649 substring false-matches fire only when `tamazia_fix_short` is empty from a fix-writer 429 (truths 36, 82); `fwName` L525 checks stale FW_NAME(40) before FW_NAME_CAT(285) (truth 83); only ~34 of ~8,959 historical audits are on the current version (truth 77); every compliance-path change mints both the client audit and the cold-email first line (truth 100).
- **Ten honest open items (Loop 3)** the rebuild must resolve in-code or with the lawyer: dimension/score locus; `compliance_enforcement`/`framework_intelligence` schemas; CAP_GATE establishment-regex breadth; the legal dataset is a draft for sign-off; US-state privacy currency; `patch-dist.js` clobbering render fixes; golden-set composition; SEO/GEO shared-state boundary; enforcement.js wiring point; re-mint batch size vs free-tier limits.

## (e) WHAT CLAUDE-MASTER-PLAN SAYS THE END STATE SHOULD BE

**The goal, stated once:** for any firm's website, attach exactly the laws that bind it — no more, no less — across UK, EU, USA (incl. state/California), and the Middle East gulf, combining universal + jurisdiction-specific + sector/sub-sector-specific laws, each carrying a live, dated, sourced enforcement signal. "Accuracy is not more laws. Accuracy is the right laws, at the right node, for the right nexus, kept current."

**The end-state model:** one Compliance Object Model + seven canonical registries. A law attaches **iff** three hold: (1) sub-sector node matches (node-only laws — POM ad ban, HFEA, ABI — only at their node); (2) `required_nexus ∈ {established_in | serves_customers_in | processes_residents_of}` genuinely holds; (3) predicates resolve (every `applies_when` producible, every `excluded_when` absent). The universal layer swaps wholesale by jurisdiction. The one law record carries: short/name/regulator_id, `sector[]`, `sub_sector[]`, jurisdiction (+region), `required_nexus`, `universal` bool, `binding_status`, `applies_when[]`/`excluded_when[]`, obligations[] (duty + evidence_type + element_checklist + severity), penalty (typical_low/high, statutory_max, basis, max_is_rare), enforcement (regulator, sanction, register_url, recent_action + date), intel (why_matters, what_assessed, current_law, most_recent_change, relevance_hook), and **provenance (source_feed, last_synced, source_url) — never render a law without it.** The five missing dimensions: sub-sector · nexus · binding-status · duty-with-evidence · typed evidence ledger over a deterministic crawl.
