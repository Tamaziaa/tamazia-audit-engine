### Two load-bearing defects found (both outside my edit scope, reported not fixed)

- **D1 — ci.yml does not parse, so the entire local tool fleet never runs on GitHub.** Line 64 (`- name: Known-bad calibration corpus (--strict: a missing checker fails the build, not just a checker that misses its fixture)`) contains an unquoted `: ` inside the `name:` value; YAML reads it as a nested mapping and aborts the document. Confirmed with a strict YAML parser and corroborated by CodeRabbit's latest review (flagged Critical, still open). Consequence: on GitHub the only checks that execute are CodeQL, Semgrep and CodeScene (separate files). ESLint, madge, jscpd, dependency-cruiser, swallow-gate, calibration, `npm test` and the catalogue compile — the whole "Local tool fleet" job the CONSTITUTION calls **live** — are silently not running in CI. This directly undercuts directives 2 and 15.
- **D2 — the runbook/charter test command is broken on the installed Node.** `node --test facts/` (written verbatim in AGENTS.md-adjacent runbooks and in the integration-verifier charter) fails on Node 26. `package.json` declares `"node": ">=24"` but the machine runs 26.3.0 and the CI step pins "Setup Node 24". This is a latent portability trap: anyone resuming per the runbook on a current Node gets a false red.

## 3. Directives with NO mechanical enforcement yet (the risk list)

These are held by good intentions, agent charters or prose only. A directive with no failing gate is, by this repo's own CONSTITUTION line 3, "theatre" until wired. Ranked by exposure.

1. **Directive 3 / 18 — caution.md per-phase walk (C-198).** The single most load-bearing anti-regression promise in the whole system is an *agent* responsibility ("the Caution warden walks every diff against every pointer"). There is no CI job that parses `caution.md`, maps pointers to gates, and fails a phase when a pointer's class reappears or when no walk was recorded. Evidence it is already slipping: zero P1/P2-dated pointers were appended despite two phases landing. **Highest risk — this is exactly the "knowledge scattered, repeats shipped" failure C-198 exists to prevent.**
2. **Directive 2 / 15 — CI actually running.** The mechanical backbone (`ci.yml`) is itself broken (D1). Every gate the CONSTITUTION marks "live in ci.yml" is currently not executing on GitHub. Enforcement that cannot start is not enforcement. Fix D1 first; nothing else on the CI side is real until then.
3. **Directive 10 — model tiering / Fable-orchestrates-only.** No log schema or check records which model ran which step. Fully unenforceable; a violation is invisible.
4. **Directive 19 — three-source validation per judgement.** Enforced only as a phase-exit conversation. No artifact per judgement, no gate. The provenance linters give one-URL-resolves, not three-source.
5. **Directive 14 — findings cleared not suppressed.** The apps run and block, but "cleared" is asserted in a commit message while the live re-review still requests changes. There is no gate asserting "zero open CodeRabbit/CodeScene actionable at merge" (branch protection could require it; it does not appear to).
6. **Directive 8 — no frivolous laws.** `client_useful` is stored but not asserted by any linter; a `false` row reaching a served cell would not fail CI.
7. **Directives 7, 9, 12(end-to-end), 16, 17** — not-yet-enforceable because their home layers (breach, llm, payload composer, render-proof, evidence) are empty scaffolds. Their CONSTITUTION gate names are honestly marked "planned P3/P4"; the risk is not deception, it is that a large fraction of the doctrine has no live gate and the schedule to build those gates is the schedule to make the directives real.

Structural observation: the facts and catalogue layers (P1/P2) are genuinely gated — one-door, swallow-gate, calibration, reference-set, schema and the linter suite all fire and catch seeded bad input. The enforcement *cliff* is at the phase boundary: everything the LLM, breach, evidence, mint and render directives promise is currently prose in CONSTITUTION.md and empty directories, plus a broken CI file that stops even the built gates from running on GitHub.
